# Repository Structure Documentation

This document describes the organized modular structure of the fym_space Jekyll site.

## Overview

The repository has been restructured for better modularity and maintainability, following Jekyll best practices:

- **_sass/** - Stylesheets organized by category (base, components, utilities)
- **_data/** - Structured data files organized by function (profile, galleries)
- **_updates/**, **_courses/**, **_education/**, **_teaching/**, **_publications/** - Markdown collections for prose-heavy content
- **_includes/** - Reusable template components (pages, scripts)
- **_layouts/** - Page layout templates
- **about.md**, **education.md**, **publications.md**, **teaching.md**, **courses.md**, **hobbies.md** - Top-level sidebar pages
- **assets/gallery/** - Image assets

## Directory Structure

### `/` (Root)
- **_config.yml** - Site configuration
- **Gemfile** - Ruby dependencies
- **.gitignore** - Git exclusion rules
- **index.md** - Homepage
- **about.md** - About page
- **education.md** - Education page
- **publications.md** - Publications page
- **teaching.md** - Teaching page
- **courses.md** - Courses page
- **hobbies.md** - Side Missions page
- **dance.md** - Dance gallery page at `/hobbies/dance/`
- **travel.md** - Travel gallery page at `/hobbies/travel/`
- **todo.md** - Personal to-do list

### `/_sass/` - Stylesheets

Organized into three categories:

#### `/base/`
Base styles and variables:
- **_variables.scss** - Color palette and design tokens (CSS custom properties)
- **_reset.scss** - Global element resets and base typography
- **_layout.scss** - Main grid layout structure

#### `/components/`
Reusable component styles:
- **_sidebar.scss** - Navigation sidebar styling
- **_about.scss** - About section layout
- **_section.scss** - Generic section containers
- **_education.scss** - Education timeline styles
- **_publications.scss** - Publication list styles
- **_courses.scss** - Course cards and reviews
- **_teaching.scss** - Teaching experience styles
- **_hobbies.scss** - Hobby cards and gallery
- **_updates.scss** - Updates section styles
- **_gallery.scss** - Gallery grid layout
- **_people.scss** - Person link styling

#### `/utilities/`
Utility and layout styles:
- **_main.scss** - Main content area padding and layout

**Import Order** (`css/style.scss`):
1. Base styles (variables, resets, layout)
2. Component styles (all UI components)
3. Utilities (layout helpers)

### `/_data/` - Site Data

Organized by content type. Keep YAML here for structured values that are not primarily prose.

#### `/profile/`
Personal profile information:
- **people.yml** - Links to collaborators and advisors

#### `/galleries/`
Gallery data:
- **dance_gallery.yml** - Dance performance images and captions
- **travel_gallery.yml** - Travel photos with descriptions

### Markdown Collections

Content that contains Markdown now lives in collection documents:
- **_updates/** - Timeline updates and news. The Markdown body is the rendered update text.
- **_courses/** - Course metadata in front matter and the overview in the Markdown body.
- **_education/** - Education metadata in front matter and the description in the Markdown body.
- **_teaching/** - Teaching metadata in front matter and the description in the Markdown body.
- **_publications/** - Publication entries, with Markdown-capable front matter for authors and venues.

### `/_includes/` - Reusable Templates

#### Root level:
- **head.html** - HTML `<head>` section (meta tags, favicon)
- **sidebar.html** - Navigation sidebar component

#### `/pages/`
Page content templates (included by top-level pages):
- **about.md** - About section with profile info
- **education.md** - Education timeline
- **publications.md** - Publications list
- **teaching.md** - Teaching experience
- **courses.md** - Course review system
- **hobbies.md** - Hobbies and side missions

#### `/scripts/`
JavaScript integration:
- **nav.html** - Navigation script for active links
- **courses.html** - Course review interactivity

### `/_layouts/` - Page Layouts
- **default.html** - Main layout template (header, sidebar, footer)

### `/assets/gallery/` - Images
Gallery images organized by category:
- Dance performance photos
- Travel photography

### `/css/` - Compiled CSS
- **style.scss** - Main SCSS file (imports all sass files)
- **style.css** - Compiled CSS (generated by Jekyll)

## Data Access Pattern

In templates, data is accessed using nested paths:

```liquid
{{ site.data.profile.people }}           <!-- Collaborators -->
{{ site.data.galleries.dance_gallery }}  <!-- Dance gallery -->
{{ site.data.galleries.travel_gallery }} <!-- Travel gallery -->

{{ site.education | sort: "order" }}     <!-- Education entries -->
{{ site.courses | sort: "order" }}       <!-- Course entries -->
{{ site.publications | sort: "order" }}  <!-- Publication entries -->
{{ site.teaching | sort: "order" }}      <!-- Teaching entries -->
{{ site.updates | sort: "order" }}       <!-- Timeline updates -->
```

## Best Practices

1. **SCSS Organization**
   - Keep base styles (typography, variables) separate
   - Group related component styles together
   - Use CSS variables (custom properties) for theming
   - Import in logical order: base → components → utilities

2. **Content Organization**
   - Use Markdown collections for prose-heavy entries
   - Keep YAML for structured lookup data, galleries, and short reusable aliases
   - Use `order` front matter to preserve display order
   - Use the supported `:emoji_name:` shortcodes in Markdown when useful; `_includes/utils/inline-markdown.html` renders them without requiring a custom plugin

3. **Data Organization**
   - Group related data by purpose (profile, content, galleries)
   - Use semantic file names
   - Keep YAML structure consistent across files
   - Document required fields in comments

4. **Template Organization**
   - Page content in `_includes/pages/`
   - Interactive scripts in `_includes/scripts/`
   - Keep includes focused and reusable
   - Use meaningful include names

## Adding New Features

### Adding a new SCSS component:
1. Create `_sass/components/_feature.scss`
2. Add styles following the existing pattern
3. Import in `css/style.scss` in the Components section

### Adding new Markdown content:
1. Create a document in the relevant collection, for example `_updates/12-new-item.md`
2. Add front matter such as `order`, labels, links, or metadata
3. Put the rendered prose in the Markdown body

### Adding new structured data:
1. Create `_data/category/feature.yml`
2. Use consistent YAML structure
3. Update data access references in templates

### Adding a new top-level page:
1. Create `_includes/pages/section.md`
2. Create a root page such as `section.md` with `layout: default`, a `permalink`, and `active_nav`
3. Include the reusable content template with `{% include pages/section.md %}`
4. Add a sidebar link in `_includes/sidebar.html`
5. Add section-specific styles to `_sass/components/` if needed

## Build & Deployment

The site is built with Jekyll using:
- Ruby 2.6+
- Bundler for dependency management
- GitHub Pages compatible (no custom plugins)

### Build locally:
```bash
bundle exec jekyll serve
```

### View site:
Visit `http://localhost:4000` in your browser

## Notes

- All SCSS files are compiled to a single `css/style.css` by Jekyll
- Data files are automatically available as `site.data` in templates
- The `.gitignore` excludes build artifacts and local IDE files
- Gemfile.lock is tracked for reproducible builds
