# Fan Jue's Personal Website

This repository contains the source for a Jekyll-powered personal website hosted on GitHub Pages:

```text
https://fyimu.github.io/fym_space/
```

The site includes research/publication pages, teaching and course notes, side-mission galleries, life-note blog posts, and a small statistics page backed by GoatCounter.

## Local Setup

Install Ruby gems:

```bash
bundle install
```

Build the site:

```bash
bundle exec jekyll build
```

Preview locally:

```bash
bundle exec jekyll serve
```

Then open:

```text
http://127.0.0.1:4000/fym_space/
```

If port `4000` is already in use, stop the old Jekyll process or run:

```bash
bundle exec jekyll serve --port 4001
```

## Repository Structure

```text
_config.yml                 Jekyll site settings and collections
_includes/                  Shared page fragments, sidebar, scripts, utilities
_layouts/                   Page layouts
_sass/                      Site styling, split by component
css/style.scss              Sass entry point
assets/gallery/             Site images and gallery photos
_data/                      YAML data for galleries, profile data, statistics
_updates/                   About-page update items
_education/                 Education entries
_teaching/                  Teaching entries
_courses/                   Course review entries
_publications/              Publication entries
_life_posts/                Life Notes blog posts
scripts/update_statistics.py GoatCounter statistics sync script
.github/workflows/          GitHub Actions workflows
```

Top-level pages such as `about.md`, `education.md`, `publications.md`, `courses.md`, `hobbies.md`, `blog.md`, and `statistics.md` define page routes and include the corresponding content.

## Editing Content

### About

Main content:

```text
_includes/pages/about.md
```

Latest updates:

```text
_updates/*.md
```

Each update has front matter such as `order`, `date_label`, optional `link`, and content.

### Publications

Each publication is stored as one Markdown file:

```text
_publications/*.md
```

The publications page renders these files through:

```text
_includes/pages/publications.md
```

### Courses

Each course review is stored as one Markdown file:

```text
_courses/*.md
```

Rendered by:

```text
_includes/pages/courses.md
_includes/scripts/courses.html
```

### Side Missions

Main side missions page:

```text
_includes/pages/hobbies.md
```

Gallery data:

```text
_data/galleries/travel_gallery.yml
_data/galleries/dance_gallery.yml
```

Gallery pages:

```text
travel.md
dance.md
```

Images live under:

```text
assets/gallery/travel/
assets/gallery/dance/
```

### Life Notes

Life-note posts are stored in:

```text
_life_posts/*.md
```

They use:

```text
_layouts/life_post.html
_sass/components/_blog.scss
```

Photos and writeups are configured in each post's front matter.

## Styling

Styles are organized by component:

```text
_sass/base/
_sass/components/
_sass/utilities/
```

The main Sass entry point is:

```text
css/style.scss
```

Shared colors are defined in:

```text
_sass/base/_variables.scss
```

## Website Statistics

The public site uses GoatCounter tracking from:

```text
_includes/head.html
```

The statistics page reads aggregate counts from:

```text
_data/statistics.yml
```

The GitHub Action:

```text
.github/workflows/update-statistics.yml
```

runs `scripts/update_statistics.py` every 6 hours and updates `_data/statistics.yml`.

## Deployment

The site is deployed with GitHub Pages from the repository. Before pushing, run:

```bash
bundle exec jekyll build
```

Then commit and push:

```bash
git add .
git commit -m "Describe the change"
git push
```

GitHub Pages will rebuild the site after the push. The statistics workflow may make additional commits when visitor counts change.

## Notes

- Do not commit API tokens, secrets, or private keys.
- Do not edit files in `_site/`; that directory is generated by Jekyll.
- Prefer editing collection files and YAML data files instead of generated HTML.
- If browser preview looks stale, hard refresh the page or restart `jekyll serve`.
